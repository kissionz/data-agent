## 总体结论

当前阶段已经完成 ChatBI 核心体验的可视化和领域状态机基座，足以作为产品评审、前后端契约拆分和后续 API 开发的起点。完整 PRD 的生产验收尚未完成，尤其是身份权限、真实数据源、Analysis IR、编译执行、审计、评测和 SLO 证明仍需后续阶段实现。

## 下一阶段验收门槛建议

1. 先定义契约包：`AnalysisIR`、`RunView`、`ResultView`、`ClarificationCandidate`、`AuditEvent`、错误码和展示状态。
2. 接入本地 API 后，所有当前 mock 场景必须迁移为 API fixtures 和 E2E 用例。
3. 每个 PRD P0 流程至少有一个可重复测试：标准查询、澄清、权限拒绝、取消、部分结果、语义发布、回放门禁。
4. 任何真实模型接入前，答案必须保留“只能引用结果集或授权知识”的测试护栏。
